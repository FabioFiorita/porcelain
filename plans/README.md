## Findings considered and rejected (so they aren't re-audited)

- **S4 — `gitCheckout` missing `--`**: REJECTED. The proposed `['checkout','--',branch]`
  fix is wrong (`--` makes git treat the operand as a *pathspec*, breaking branch
  checkout), and a real git ref cannot begin with `-` (so the argument-injection it
  guards against is unreachable, and the input comes from the real branch list).
  Net negative; not worth doing.
- **P3 — module caches never evict** (`flowCache` etc.): NOT WORTH DOING standalone.
  One entry per repo, one repo per window, overwrite-on-same-key, freed on window
  close. Fold an LRU in only if touching those caches for another reason.
- **CORRECTNESS-02 — cross-process lost-update on the agent channels**: DEFERRED. The
  `audit` skill already documents this as an accepted, rare, low-stakes tradeoff
  (atomic writes both sides; the watcher re-syncs). A cross-process lock is MED-risk
  (Electron ↔ bare-node) and would re-litigate a documented decision — not worth the
  risk this round. Revisit only if a real lost-write is observed.
- **PERF-B4/B5 — O(F·I·U) import resolution in the flow build**: DEFERRED. A clean
  algorithmic win but *bounded by the 200/60 caps* (the perf audit's own words: "a
  clean win, not a cliff"), and it touches `flow.ts` — the single shared flow resolver
  — where a subtle path-matching change risks mis-grouping. Defer until there are
  characterization tests for `flow.ts` or a measured need.
- **B5 — staging hooks skip invalidate / don't surface error on failure**: DEFERRED.
  Low impact (on a staging failure nothing changed, so the skipped invalidate is
  benign), and surfacing the error ripples the hook return shape into the callers for
  a rare path. Not worth the ripple this round.
- **S2 — fs procedures accept absolute paths**: BY DESIGN (the trusted-renderer model;
  CSP + no remote content). Not a finding; the real residual (explore-reader
  containment) is a small, low-priority hardening noted but not planned.
