## Findings considered and rejected

- **Markdown XSS**: not a finding — `react-markdown` is used without `rehype-raw`,
  so raw HTML in repo markdown is never rendered. (`markdown-view.tsx`)
- **Arbitrary local-path reads in fs procedures** (`readFile`/`readDir`/`writeTextFile`/
  `revealInFinder` accept absolute paths): by-design. Porcelain is a local file
  viewer with a context-isolated, trusted renderer and no remote content; reading
  local paths is the product. Not a vulnerability under that threat model.
- **Unbounded in-memory caches** (`flowCache`, `visibleFilesCache`, `fileListCache`
  in `api.ts`/`git.ts`): bounded by repos opened per window (≈1), not worth evicting.
- **esbuild dev-only advisories** (2 high / 1 low, transitive via `vite`): build-time
  only, no prod advisories, one isn't even reachable (Deno-specific) and the other is
  Windows-only dev-server. Fix is a routine `vite` bump; not worth a dedicated plan.
  Do it opportunistically: `pnpm up vite` (verify esbuild resolves to ≥0.28.1, then
  re-run the full gate).
- **`viewer.tsx` `tab.kind` if-chain vs `never`-checked switch**: minor style nit
  against the exhaustive-switch rule; not worth a plan on its own.
