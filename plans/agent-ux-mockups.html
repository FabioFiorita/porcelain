Agent UX — direction mockups

Not shipped UI. A proposal for the two problems that make the Agent tab scary: (1) no mental model for keep vs close threads, (2) the live surface is thin compared to the terminal CLIs you already trust. Open this file in a browser; iterate from here.

Facts that should calm the “Mac off” fear: threads are daemon-owned files under ~/.porcelain/agent-threads/ on the machine running the daemon. If the window is pointed at the Linux / Beelink daemon, turning the Mac off does not kill the turn — the Linux process keeps working. Closing the Mac app only detaches the viewer. Delete is the only intentional kill of a thread transcript. Today’s problem is UX: trash looks like the only way to leave a thread, so people delete to “clean up.”

Why this matters

  • You should never need to open the viewer to know if something is still chewing tokens on Linux.
  • Subagents become first-class rows, not anonymous “Task” tools.
  • Environment line (“Daemon: beelink”) ties Agent UX to the env-switch trust problem.

4. Token bloat — separate workstream

The wrapper injects a Porcelain preamble every turn and normalizes CLI streams into a reduced timeline. That can inflate context vs a raw terminal session (skills list, system append, tool schema shape). This is not fixed by UI alone — needs a measured “tokens per turn” comparison (terminal vs Agent tab) and a thinner preamble / less re-injection. Track as its own investigation; do not claim “fixed” without numbers from the same prompt on both surfaces.
