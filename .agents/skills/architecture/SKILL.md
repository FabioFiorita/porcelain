# Porcelain architecture

## Stack

| Area | Decision |
|---|---|
| Shell | Electron via **electron-vite**, React 19, TypeScript (strict) |
| UI | **shadcn/ui on Base UI** (`@base-ui/react`, not Radix) + Tailwind CSS v4, `base-nova` preset, Geist font, dark mode default |
| Client architecture | **Porcelain's own conventions** (see "Client architecture" below): tab-store routing, domain data hooks, one public component per file |
| Client state | **zustand** — small stores per concern; no other state libraries |
| Git backend | Shell out to `git` CLI from the main process; parse porcelain-format output; no git libraries |
| Per-repo config | App-side JSON store under `~/Library/Application Support/porcelain`, keyed by repo path; never write into work repos |
| Package manager | **pnpm** |
| Lint/format | **Biome** (no ESLint/Prettier) |
| Tests | **Vitest** (unit/component) + **Playwright** (Electron e2e, not yet wired) |

## App shell

- No repo open → full-screen `welcome` component (repo picker); shell renders only once a repo is selected.
- Layout: shadcn `sidebar` left + `SidebarInset` holding the tab bar and viewer + the right Quick Access sidebar. NO embedded terminal — Porcelain is a *companion to* the user's terminal (Ghostty/Warp), never a terminal itself; don't reintroduce one.
- **One repo per window** — window state is scoped to a single repo/worktree.
- Shell components live in `src/renderer/src/components/shell/` (`app-shell`, `app-sidebar`, `right-sidebar` + its `pinned-group`/`quick-commands-group`/`commit-group` sections, `file-tree` + `tree-node`, `tab-bar`, `viewer`); the viewer's pieces (`file-content`, `text-file-view`, `editor-source`, `source-view`, `source-context-menu`, `find-bar`, `use-path-actions`) live in `components/viewer/`; stores: `stores/tabs.ts`, `stores/repo.ts`.
- File tree: lazy per-directory reads (`readDir` on expand), nothing indexed up front; built from shadcn `SidebarMenu` + `Collapsible` + `ContextMenu`.
- Glass look: `vibrancy: 'under-window'` + `titleBarStyle: 'hiddenInset'` + transparent backgroundColor in main; dark theme `--background`/`--sidebar` have alpha; `.app-drag`/`.app-no-drag` classes mark window-drag chrome. Sidebar header has `pl-[4.75rem]` for traffic lights; the `ProjectSwitcher` is left-aligned right after them (same row, truncates if long) — NOT centered (tried, looked like a button floating mid-header). The three chrome headers (left sidebar, main `TopBar`, right sidebar) are all `h-12` + `items-center` so their content shares one baseline; `trafficLightPosition: { x: 19, y: 18 }` in main centers the macOS buttons in that 48px bar (don't change one header height without the others, or the lights drift).
- Sidebar is drag-resizable: `sidebarWidth` in `stores/preferences.ts` (clamped 180–520) feeds `--sidebar-width` on `SidebarProvider`; handle in `sidebar-resize-handle.tsx`. During the drag the handle writes the CSS variable straight onto the `[data-slot="sidebar-wrapper"]` element and commits to the store only on mouseup — store writes per mousemove re-render the whole app.
- Right "Quick Access" sidebar (`right-sidebar.tsx`): a SECOND `SidebarProvider` nested inside `SidebarInset` (controlled via `rightSidebarOpen` preference; `shortcut="."` — the provider gained a `shortcut?: string | null` prop so the two providers don't both grab Cmd+B). `TopBar` lives inside the inner provider, so the left sidebar's toggle/state are captured by `RepoShell` (rendered between the providers, where `useSidebar()` is the outer one) and passed down as props. The two `TopBar` toggles use DELIBERATELY DIFFERENT icons — left = `PanelLeft` (file panel), right = `Zap` (quick actions) — never mirror-image panel icons. Sections follow the left sidebar's active tab (`sidebarTab` in `stores/preferences.ts`, the left `Tabs` is controlled by it), each in its own file (`pinned-group.tsx`, `quick-commands-group.tsx`, `commit-group.tsx`): Files → `files-quick-access.tsx` (a vertical split — `pinned-group.tsx` rendering `TreeNode`s from `pinnedEntries` on top, a drag-resizable divider, then the `notes-card.tsx` quick-notes editor below; see "Notes card"); Changes + History → Quick commands; Changes only → Commit composer (`gitCommitConventions` = `parseConventions` in `src/main/conventions.ts` over the last 200 log subjects → type/scope chips + message; chips are STRICTLY history-derived — top 8 types/scopes by frequency, `DEFAULT_COMMIT_TYPES` only when the history has no conventional commits). "Commit" runs `gitCommit` (in git.ts: `git commit -m` on the **staged** changes only — no `git add`; git stderr rethrown so the mutation error renders inline); a separate "Stage all" button runs `gitStageAll` (`git add -A`) via `useStageAll` (inline confirmation/error in the composer); per-file staging lives in the **left** Changes list — `parseStatus` derives `staged`/`unstaged` booleans from the porcelain XY columns onto `ChangedFile` (flow through `FlowFile`), `changes-list.tsx` shows a dot per row (emerald = staged, amber = partially staged `MM`) and a right-click menu with Stage (`gitStageFile` = `git add -- path`) / Unstage (`gitUnstageFile` = `git restore --staged -- path`) via `useFileStaging`; all three staging hooks `utils.gitFlow.invalidate()` so the dots update. `useCommit` in `hooks/use-commit.ts` owns the gitFlow/gitLog/conventions invalidation. Quick commands run a WHITELIST (`QUICK_COMMANDS` in git.ts) via `useQuickCommand` — stdout+stderr combined (push logs to stderr), shown inline in the section, errors in red; the hook does a blanket `utils.invalidate()` after every run since pull/stash change everything (the one sanctioned blanket invalidation).
- Pinning: per-repo `pinnedPaths` in the config schema, tree context menu Pin/Unpin (`pinPath`/`unpinPath`, `DirEntry.pinned` flag); invalidate `readDir` + `pinnedEntries` together.
- Notes card: per-repo `notes` string in the config schema (`notesFor`/`withRepoNotes` in `repo-config.ts` → `repoNotes` query + `setRepoNotes` mutation → `use-repo-notes.ts` hook). A TipTap WYSIWYG editor (`notes-card.tsx`) — markdown rendered live as you type, persisted as a markdown string via `tiptap-markdown`, with EditorSource-style debounced autosave (800ms + flush on unmount). SCOPED EXCEPTION to the "no CodeMirror/Monaco, file viewer stays plain-textarea" rule: TipTap is allowed for the notes card ONLY (it's a companion surface, not source editing). StarterKit v3 bundles Link (configure via StarterKit, don't add `extension-link` — double-registration throws); Placeholder is separate. Toolbar is shadcn `Toggle`/`Button`/`DropdownMenu`. `editor.storage.markdown` is typed via a `declare module '@tiptap/core'` augmentation (so `@tiptap/core` is an explicit devDependency) — no cast. The resizable divider is `NotesResizeHandle` (`sidebar-resize-handle.tsx`, vertical sibling of the width handles): drives `--notes-height` on `[data-slot="files-quick-access"]` live, persists `notesHeight` (preferences store, clamped 100–600). GOTCHA: ProseMirror calls `document.elementFromPoint` on mount — jsdom lacks it, so `test-setup.ts` stubs it (returns null) alongside `matchMedia`.
- Tree multi-select: cmd/ctrl-click toggles selection (`stores/selection.ts`); context menu shows "Hide N items" when a selection exists; batch runs hide mutations then one invalidate.
- Folder hiding: hidden paths are absolute, any depth (files or dirs), stored per repo in the app config (`src/main/repo-config.ts` pure logic + `config-store.ts` persistence at `userData/config.json` via the generic `json-store.ts` — atomic tmp+rename writes, corrupt files backed up to `.corrupt-*`, and `updateConfig(mutate)` serializes read-modify-write so concurrent mutations never drop; read-only callers still use `loadConfig`). `readDir` filters them in the MAIN process; `showHidden` mode returns them flagged (`DirEntry.hidden`) and the UI dims them. Hide/unhide via right-click context menu; eye toggle in sidebar header. Tree refresh = `treeVersion` bump in `stores/repo.ts`.
- Dev/prod isolation: in dev, main sets `userData` to `porcelain-dev` before any config read and `seedDevConfig` (`src/main/dev-config.ts`) seeds empty recents with `~/Code/porcelain-playground` — `pnpm dev` always opens the playground, never the user's repos.
- Welcome screen lists recent repos (max 10, `recentRepos` query; `openRepoPath` mutation). On startup the last repo auto-opens (`restoreLastRepo` in `stores/repo.ts`); welcome only shows if none/missing.
- Flow-ordered review: pure logic in `src/main/flow.ts` — `DEFAULT_LAYERS` ordered Pages→…→Data→Tests; `layerFor` uses deepest-matching-segment (not first-match; filename patterns like `\.spec\.`/`\.stories\.` naturally beat the containing directory — no special cases); `parseImports` + `resolveImport` (relative + alias-suffix heuristic) connect changed files; `buildFlow` groups in layer order. `gitFlow` procedure reads ≤200 changed files (≤1MB each) for edges. Per-repo `layers` override: edited in the settings dialog's Review flow section (`components/settings/settings-dialog.tsx` + `flow-layers-section.tsx`, gear in the sidebar footer; the dialog nests a `SidebarProvider` with `shortcut={null}` and a General section for diff/markdown prefs) → `repoLayers`/`setRepoLayers` procedures (regex validated in zod) → `layersFor`/`withRepoLayers` in `repo-config.ts`. Changes list renders these groups with `connects` shown as "→ file" lines.
- Worktrees/branch: `useBranch` (rev-parse, 5s poll) + `useWorktrees` (`worktree list --porcelain`, parser in diff.ts); `WorktreeSwitcher` in the sidebar footer (shadcn `dropdown-menu`) calls the repo store's `switchTo` — the ONE place repo switching lives (closes all tabs, then opens the path; `ProjectSwitcher` uses it too). GOTCHA: Base UI requires `DropdownMenuLabel` (GroupLabel) inside `DropdownMenuGroup` — outside one it throws MenuGroupContext missing.
- History: `gitLog`/`gitCommitFiles`/`gitCommitDiff` (parsers `parseLog`/`parseNameStatus`); sidebar History tab opens `commit` tabs rendered by `CommitView` (file list + per-file diff via shared `HunksView`/`DiffModeToggle`).
- Sidebar tab bar = floating glass pill (`TabsList`, `rounded-full bg-sidebar-accent/40 backdrop-blur-xl`): a STATIC `shrink-0` header, NOT sticky. The scroll lives on each `TabsContent` (`min-h-0 flex-1 overflow-auto`) with `SidebarContent` set to `overflow-hidden`, so the tree scrolls *beneath* the pill and rows never ghost through the glass (the old `sticky top-2` let content bleed above/through it). The `Tabs` wrapper is a `@container flex h-full min-h-0 flex-col` and trigger labels are `hidden @[17rem]:inline` (icons-only when narrow). GOTCHA: don't put `@container` on the TabsList itself — inline-size containment collapses its `w-fit` width.
- Shortcut hints use the shadcn `Kbd` component (`ui/kbd.tsx`): TopBar toggle tooltips (⌘B / ⌘.), sidebar tab tooltips (⌘1/⌘2/⌘3), context menus (⌘X/⌘C/⌘V, ⌘W on tab Close), empty-viewer ⌘P hint, editor Unsaved chip (⌘S).
- Shortcuts: Cmd+W intercepted in main (`before-input-event`, blocks default window-close) → `appEvents` tRPC subscription (the only subscription pattern, consumed in `hooks/use-app-events.ts`, mounted in `AppShell`) → renderer closes active tab or window; Ctrl+Tab/Ctrl+Shift+Tab cycle tabs (`use-app-shortcuts.ts`, keyboard-only). Shortcut placement policy is tiered and deliberate — see "Client architecture".
- Tab bar: horizontal shadcn `ScrollArea` (the ui component takes an `orientation` prop); titles truncate at `max-w-40`; middle-click closes; right-click context menu = Close / Close Others / Close to the Left / Close to the Right / Close All (bulk actions in `stores/tabs.ts`, anchor tab activated when the active tab is closed).
- Crash visibility: `ErrorBoundary` wraps the app in `App.tsx` (render crashes show the stack, not a blank window); in dev, main pipes renderer `console-message` and `render-process-gone` to stdout — Claude reads them from the `pnpm dev` log to self-verify.
- Vite: `optimizeDeps.entries` covers `src/**/*.{ts,tsx}` so every `@base-ui/react/*` entry is pre-bundled. A dep discovered lazily mid-session re-optimizes, loads a second React copy, and crashes with "Invalid hook call".
- `readFile` returns a `FileView` union (text | image dataUrl | binary size) — viewer renders each.
- Text file tabs render `TextFileView` (`components/viewer/text-file-view.tsx`, header = relative path + actions). Text files are ALWAYS editable (no edit button/state): `EditorSource` (`components/viewer/editor-source.tsx`) = plain textarea over a Shiki-highlighted backdrop with a line-number gutter (textarea text transparent + `caret-foreground`, aria-hidden mirror of memoized `CodeLine`s; keep textarea `pl-14` aligned with the gutter or the caret drifts). Both layers live in ONE `overflow-auto` scroll container — the textarea sizes to content via `field-sizing-content` so the container (not the textarea) scrolls, and the backdrop is `absolute inset-0` behind it. They CANNOT desync because there's a single scroll; the old design scrolled the textarea and synced the backdrop in `onScroll`, which lagged and left the native selection over the wrong lines after a scroll. Autosave: 800ms debounce after last keystroke + flush on unmount + Cmd+S; status chip bottom-right (Unsaved ⌘S / Saving… / error); saves via `useWriteTextFile` (`hooks/use-files.ts`, which owns the readFile + gitFlow + gitDiffFile invalidation); first keystroke pins a preview tab. Files over 5000 lines (`EDITABLE_MAX_LINES`) fall back to the read-only virtualized `SourceView`; markdown reader mode is read-only (source mode edits). The editor's own context menu covers Cut/Copy/Paste/Find references (textarea selection) + path actions; capture the selection in `onOpenChange` into state — nothing re-renders on text selection, so reading it at render time leaves items stale/disabled. Menu items show `Kbd` shortcuts inside `ContextMenuShortcut`. `FileContent` is keyed by path in `Viewer` so edit state can't leak across tabs. Deliberately NOT an editor: no CodeMirror/Monaco, no LSP.
- Viewer context menu (`components/viewer/source-context-menu.tsx`): Copy / Find references (selection) / Copy path / Copy relative path / Reveal in Finder (`useRevealInFinder` → shell.showItemInFolder). GOTCHA: the ui `ContextMenuTrigger` defaults to `select-none` — pass `select-text` or text selection silently breaks. Selection is captured in the menu's `onOpenChange`. The edit-mode textarea gets its own Cut/Copy/Paste menu (clipboard API + manual splice).
- Find references: `searchText` procedure = `gitGrep` (`git grep -n -I --untracked --fixed-strings`, ≤500 matches, exit 1 → []; parser `parseGrep` in diff.ts). Results render in a `search`-kind tab (`SearchView`); clicking a match opens the file tab with `Tab.line`, which `VirtualRows`' `scrollToLine` centers and `SourceView` highlights. `openTab` on an existing id updates `line` so repeated jumps work.
- Markdown files get a Reader/Source toggle (private `MarkdownModeToggle` in `components/viewer/text-file-view.tsx`; `markdownMode` preference, reader default). Reader = `MarkdownView` (react-markdown + remark-gfm, `prose prose-invert` via `@tailwindcss/typography`, loaded with `@plugin` in main.css); links get `target="_blank"` so main's `setWindowOpenHandler` opens them externally — gated by `isSafeExternalUrl` (`src/main/external-url.ts`, http/https/mailto allowlist; extend `ALLOWED_PROTOCOLS` deliberately, never drop the guard). Reader view is NOT virtualized — markdown only; never route code files through it.
- UI preferences (diff mode, markdown mode, sidebar tab, right/left sidebar open + widths) persist via `zustand/middleware` `persist` to localStorage (`porcelain-preferences`). Both sidebars drag-resize (`SidebarResizeHandle`/`RightSidebarResizeHandle` write `--sidebar-width` directly during drag, commit on mouseup).
- Git: pure parsers in `src/main/diff.ts` (porcelain-z status, unified diff → `DiffHunk[]`, synthesized add-diff for untracked) + shell-out in `src/main/git.ts`. Sidebar has Files/Changes tabs (shadcn `Tabs`); changes list opens `diff` tabs; `DiffView` (`components/git/diff-view.tsx`) renders unified or split via `HunksView` (`hunks-view.tsx`, split rows paired del↔add in `toSplitRows`) and `DiffModeToggle` (`diff-mode-toggle.tsx`, preference in `stores/preferences.ts`) — both shared with `CommitView`.

## Packaging, signing, updates

- `electron-builder.yml`: appId `com.fabiofiorita.porcelain`, productName Porcelain, mac targets dmg + zip (arm64; the ZIP is what electron-updater downloads), hardened runtime, signs with the keychain "Developer ID Application" identity automatically. `notarize: false` until notarytool creds are configured (`APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, `APPLE_TEAM_ID` env vars — then flip to true).
- Scripts: `pnpm dist` = typecheck + build + signed DMG/ZIP into `dist/` (no publish); `pnpm release` = same + publish to GitHub releases `fabiofiorita/porcelain` (requires `GH_TOKEN` env, bump `version` in package.json first).
- Auto-update: `electron-updater` wired in `src/main/updater.ts` — no-op unless `app.isPackaged`; checks on launch + every 4h, auto-downloads, installs on quit. State flows: updater events → `update-status` AppEvent → renderer invalidates the `updateStatus` query → `UpdateButton` in TopBar shows "Update to vX" when downloaded → `installUpdate` mutation = quitAndInstall.
- Icons: `build/icon.png` is the 1024 master (832px squircle artwork centered with transparent padding); `icon.icns` regenerated via iconutil, `icon.ico` via ImageMagick, `resources/icon.png` 512 (linux window icon), `src/renderer/src/assets/logo.png` 512 for in-app branding. Regenerate all from a new 1024 source with ImageMagick + iconutil.

## Repo facts

- Aliases `@renderer/*` → `src/renderer/src/*` and `@main/*` → `src/main/*`, defined in FOUR places that must stay in sync: `electron.vite.config.ts`, `tsconfig.web.json`, root `tsconfig.json` (needed by the shadcn CLI), `vitest.config.ts`. `@main` imports in the renderer are **type-only** (`import type`) — never runtime-import main code; esbuild erases them so nothing leaks into the bundle.
- shadcn components: `src/renderer/src/components/ui/` (excluded from Biome); add via `pnpm dlx shadcn@latest add <name>`. Base UI uses `render` prop, not Radix's `asChild` — see `.agents/skills/shadcn/rules/base-vs-radix.md`.
- Tailwind/theme entry: `src/renderer/src/assets/main.css` (imports `shadcn/tailwind.css` and Geist).
- Main process = OS/git/fs access. Renderer = pure UI, no Node APIs.
- **Data fetching = TanStack Query via domain hooks** (`@trpc/react-query@10` + `@tanstack/react-query@4`, versions paired with tRPC v10). Components NEVER import `@renderer/lib/trpc` (Biome `noRestrictedImports` override enforces this in `components/**`); all server access goes through hooks in `src/renderer/src/hooks/use-<domain>.ts` that wrap `trpc.*.useQuery/useMutation` and OWN their post-mutation invalidation. Provider in `lib/query.tsx` (staleTime 30s, no focus refetch). The vanilla `trpcClient` is sanctioned only in zustand stores (`stores/repo.ts`) and the `use-app-events.ts` subscription hook. Never ad-hoc useEffect+useState fetching; never prefix invalidation/prefetch with `void` — use `async`/`await` or `await Promise.all([...])`.
- **Virtualized code rendering**: file viewer and diff view render through `VirtualRows` (`components/viewer/virtual-rows.tsx`, @tanstack/react-virtual, fixed 20px rows) — NEVER render all lines of a file; Shiki tokenizes only mounted rows. Diff hunks flatten to a `DiffRow` union (`header | unified | split`) before virtualization.
- Query freshness: fs-backed queries keep the 30s default staleTime; git-backed queries are live (`gitFlow`: staleTime 0 + 3s poll while mounted; `gitDiffFile`: staleTime 0, keepPreviousData). The 3s `gitFlow` poll is cheap because main memoizes the flow result on a status+numstat+layers key (`flowCache` in `api.ts`) — changed-file contents are re-read only when the working tree actually changes.
- Hover prefetch: changes-list rows prefetch `gitDiffFile` (staleTime 2s) and file-tree rows prefetch `readFile` on mouseenter, so opening feels instant.
- **Syntax highlighting = Shiki**, theme `dark-plus`, singleton in `lib/highlight.ts` (`languageFor(path)` ext→lang map); `CodeLine` + `useHighlighter` in `components/viewer/code-line.tsx` render per-line tokens, shared by file viewer and diffs.
- **IPC = tRPC via `electron-trpc`** (the only IPC pattern). PINNED to tRPC **v10** (`@trpc/*@^10`): electron-trpc 0.7 reads v10 internals (`_def.query`); tRPC v11 silently breaks every call with NOT_FOUND. Don't upgrade until electron-trpc supports v11 (watch `electron-trpc-experimental`). Renderer client uses `createTRPCProxyClient`. Router + procedures in `src/main/api.ts` (zod inputs, exported `AppRouter` + shared types like `RepoInfo`/`DirEntry`); preload calls `exposeElectronTRPC()`; renderer client in `src/renderer/src/lib/trpc.ts` (`import type { AppRouter }` from main — type-only import, no runtime coupling). Never use raw `ipcMain`/`ipcRenderer` channels, never cast (`as unknown as` is banned repo-wide).

- Tabs: single-click in the tree opens a *preview* tab (`Tab.preview`, italic title) that the next preview replaces; double-click (tree or tab) or entering edit mode pins via `pinTab`. Cmd+1/2/3 switch the sidebar tabs (`use-app-shortcuts.ts`).
- Cmd+F find-in-file: `FindBar` (`components/viewer/find-bar.tsx`, line-based case-insensitive matches, Enter/Shift+Enter cycle, Esc closes); current match feeds `SourceView`'s `highlightLine`. It is a FLOATING overlay (`absolute right-3 top-3 z-30` rounded card over the code), not a stacked bar — `TextFileView` wraps the content area in `relative` and renders it inside, so it never adds a third header row. Source rows never wrap (`whitespace-pre` in `CodeLine`, `w-max min-w-full` rows in `VirtualRows` → horizontal scroll); wrapping inside 20px virtual rows overlaps lines.
- `readFile` stats first and returns `{type:'too-large'}` above `MAX_READ_BYTES` (10MB, `src/main/read-limits.ts`) — never read the bytes of oversized files.
- Sidebar header = `ProjectSwitcher` dropdown (recent repos + open dialog; switching clears tabs). File rows render colored per-filetype icons from `components/viewer/file-icon.tsx` (`FileTypeIcon`/`FolderIcon`); `.DS_Store` is filtered in `readDir`.
- Git suggestions: pure `parseSuggestions` in `src/main/suggestions.ts` (status --porcelain=v2 --branch + stash list) → `gitSuggestions` query → sparkle rows above Quick commands in the right sidebar (5s poll).
- Theme: shadcn preset `b2D0xPJT8` (luma/Geist Mono/small radius), retinted to a **neutral graphite** accent — `--primary`/`--ring`/`--sidebar-primary`/`--chart-*` are monochrome (no colored brand accent), the single theme for v1. There is NO theme picker (a blue option + Theme settings tab were built and then cut — user wanted one dark theme to ship; see decision log). Re-applying a preset overwrites `ui/` AND the preset color block — restore the sidebar `shortcut` prop, ScrollArea `orientation` prop, the dark alpha tokens (`--background`/`--sidebar`), the transparent `body` (no `bg-background`), and re-apply the neutral accent in main.css afterwards; `apply` needs a temporary stub `vite.config.ts` to pass framework detection. Static accent colors live ONLY in the token block — components use `--primary`/tokens. Multi-hue *semantic* colors are deliberate and stay literal: git status (M amber / A green / D red / R sky), per-language file icons, and the Files/Changes/History category icons encode meaning, not the brand accent. To add themes later: re-introduce `[data-theme='x']` token blocks + a `data-theme` attribute on `<html>` driven by a preference.
- Cmd+P file finder: `file-finder.tsx` (shadcn `command` dialog, `shouldFilter={false}`); search runs in MAIN via `searchFiles` — `git ls-files` (stale-while-revalidate cache in `git.ts`, warmed on repo open) filtered by hidden paths (pure `visibleFilePaths` in `repo-config.ts`, tested; memoized per repo+hidden set in `api.ts`), ranked by the pure subsequence scorer in `src/main/fuzzy.ts` (contiguity + basename bonuses; supports typing the end of a name). Top 50 results. Renderer debounces keystrokes 100ms, shows "Searching…" while fetching, and resets the query when the dialog closes.

## Client architecture

The renderer has exactly one architecture; every feature follows it. Layering, top to bottom:

```
main process (src/main/api.ts procedures, pure logic in own modules)
  → lib/trpc.ts (ONE client; import restricted to hooks/ and stores/)
    → hooks/use-<domain>.ts (domain data hooks: queries, mutations, invalidation)
      → components/<area>/*.tsx (UI only; consume hooks + stores)
stores/ (zustand: client-only state — tabs, repo, preferences, selection)
```

### Routing — the tabs store IS the router

There is no URL routing and no router library; the active tab's `(kind, path, line)` in `stores/tabs.ts` is the entire navigation state. A screen = a `TabKind` (`file | diff | commit | search`). `Tab.path` is overloaded per kind (file path / file path / commit hash / query — documented on the type). Tab ids are ALWAYS built with `tabId(kind, key)` from `stores/tabs.ts` — never hand-build `"diff:..."` strings. `Viewer` (`components/shell/viewer.tsx`) dispatches kind → view with an exhaustive `switch` (no default; the annotated return type turns a missing case into a compile error). Preview semantics: single-click opens a preview tab (italic) that the next preview replaces; double-click, editing, or a non-preview re-open pins (`pinTab`).

**Recipe — adding a new screen/tab kind** (e.g. `blame`), in order:
1. Pure logic in its own `src/main/<thing>.ts` module with a `<thing>.test.ts` next to it.
2. Procedure on the router in `src/main/api.ts` (zod input; `AppRouter` updates automatically).
3. Hook in `src/renderer/src/hooks/use-<domain>.ts` wrapping the procedure (new file only for a genuinely new domain).
4. `'blame'` added to `TabKind` in `stores/tabs.ts`.
5. View component `components/git/blame-view.tsx` — one public component, takes its key as a single prop (`hash`/`path`), reads data via the hook.
6. Opener calls `openTab({ id: tabId('blame', key), ... })`.
7. New `case` in `Viewer`'s switch (the compiler forces this).
8. Keyboard binding in `use-app-shortcuts.ts` if needed.

Repo switching is one store action: `useRepoStore.switchTo(path)` (closes all tabs, opens the repo) — used by `ProjectSwitcher` and `WorktreeSwitcher`; never clear tabs ad hoc. The right sidebar deliberately follows `preferences.sidebarTab` (the LEFT sidebar's tab), not the active main tab — two parallel nav axes by design.

### Data hooks (`src/renderer/src/hooks/`)

- One module per domain: `use-repo`, `use-files`, `use-git-flow`, `use-diff`, `use-history`, `use-commit`, `use-worktrees`, `use-repo-layers`, `use-search`, `use-updates`, `use-app-events`. Thin declarative wrappers — no business logic.
- Query options (enabled guards `repo !== null`, `staleTime`, `refetchInterval`, `keepPreviousData`) live in the hook, not the component.
- **Hooks own invalidation**: each mutation hook lists its targeted invalidations in `onSuccess` (`Promise.all`). The ONLY blanket `utils.invalidate()` is `useQuickCommand` (pull/stash change everything — documented escape hatch). Hover prefetch is a hook too (`useDiffFilePrefetch`, `useReadFilePrefetch`).
- `use-app-events.ts` holds the single tRPC subscription (vanilla client), mounted once in `AppShell`.
- Enforced: a Biome `overrides` block in `biome.json` makes importing `@renderer/lib/trpc` from `components/**` a lint error.

### State placement — one rule per kind of state

- Server/git/fs state → TanStack Query via domain hooks, nowhere else.
- Cross-component UI state → a zustand store in `stores/`, one file per concern; components subscribe with fine-grained selectors (`useXStore((s) => s.field)`) directly at the leaf — no prop-drilling stores (the one exception: `LeftSidebarHandle` drilled `RepoShell`→`TopBar`, forced by nested SidebarProviders).
- Prefs that survive reload → the single persisted `preferences` store (localStorage `porcelain-preferences`). Nothing else persists.
- Everything else → component-local `useState`. Never `useState` for state another component reads.
- Store actions may call other stores via `useXStore.getState()` (e.g. `switchTo` → `closeAllTabs`); components use hooks.

### Component authoring

- **One public component per file**; filename = kebab-case of the export. Private module-scoped subcomponents are allowed only when tightly bound to the export (a row, a toggle used once); an independent feature (own queries/state) gets its own file. Allowed co-location exceptions: inseparable variant pairs (`file-icon.tsx`, `sidebar-resize-handle.tsx`), a component plus its companion hook (`code-line.tsx` + `useHighlighter`), and mutually recursive components (`tree-node.tsx`: `TreeNode`/`DirNode`).
- Named exports only (`export function PascalCase()`); the sole default export is `App`. Class components only where React requires it (`ErrorBoundary`).
- Explicit return types: `React.JSX.Element` (`| null` when conditionally empty); handlers `void`/`Promise<void>`.
- Props typed INLINE in the destructuring parameter. A named `XProps` interface is allowed ONLY for generic components (`VirtualRowsProps<T>`). Domain/data types may be named interfaces.
- Handlers named by intent (`run`, `save`, `switchTo`, `select`) — never `handleX`; callback props use React-native `onX` names.
- Composition: plain prop-driven components + zustand; `children`-wrapper components for menu/boundary shells; render-prop only for generic virtualized lists (`VirtualRows`); Base UI's `render={<.../>}` slot to merge shadcn triggers. NO app-authored React context, no boolean-prop variant proliferation. (The former vercel-composition-patterns pointer is retired — these rules replace it.)
- Pure single-file helpers stay module-scoped at the top of the file; pure shared helpers go in `@renderer/lib/` with a test; reused stateful logic is a `useX` hook returning named callbacks. Derive, don't store, computed values.
- Always `cn()` for conditional/merged classNames — no template-string class concatenation.

### Keyboard shortcuts — tiered ownership (deliberate, don't "centralize")

1. Main-process `before-input-event` ONLY to override an Electron/OS default (Cmd+W).
2. App-global bindings that act on stores → `use-app-shortcuts.ts` (Ctrl+Tab, Cmd+1/2/3).
3. A shortcut toggling one component's local state registers its own window listener in that component (Cmd+P in `file-finder`, Cmd+F in `text-file-view`).
4. Focused-element shortcuts as element `onKeyDown` (Cmd+S on the editor textarea).
5. Sidebar toggles via the `SidebarProvider` `shortcut` prop (Cmd+B / Cmd+.).

### Testing

- Pure logic (main-process parsers, lib helpers, zustand stores) → unit tests next to source (`foo.test.ts`), named after the unit. This is where most coverage lives — keep logic pure and main-side.
- Component tests (`foo.test.tsx`, next to source) mock the **domain hooks**, never the tRPC proxy: `vi.mock('@renderer/hooks/use-history', ...)` returning plain objects. Exemplars to copy: `components/git/history-list.test.tsx`, `components/git/changes-list.test.tsx`. Shape mock data with `@main` types so drift breaks the build.
- `lib/trpc.ts` CANNOT be imported under vitest — `ipcLink()` throws without the preload-injected `electronTRPC` global. The hooks layer exists partly so component tests never load it.
- Setup: `src/test-setup.ts` (registered in `vitest.config.ts` `setupFiles`) wires jest-dom matchers, an explicit `afterEach(cleanup)` (globals are off — tests import from `'vitest'`), and a `window.matchMedia` stub (required by any test mounting `SidebarProvider`). jest-dom types load via the triple-slash reference in `env.d.ts`.
- Reset zustand between tests with `useXStore.setState(...)` in `beforeEach`. No snapshot tests.

## Conventions

- **shadcn primitives only**: never hand-roll a primitive (sidebar, tabs, dialog, tree, …); search shadcn/registries first; building a new primitive requires user approval.
- Strict TS, no `any`, no `as unknown as`, no dead code, no commented-out code.
- Conventional Commits (`feat:`, `fix:`, `chore:`, `refactor:`, `test:`, `docs:`).
- Verification gate before any commit: `pnpm lint && pnpm typecheck && pnpm test && pnpm build` must all pass.
- **Decision log** (CLAUDE.md): dateless bullets, append-only, position = chronology (git holds timestamps). Terse style — bold lede for a major decision, GOTCHA/file pointers inline. Every entry also lands in its home skill (hard rule 4): the log records *why*, the skill states *what* — the skill is the source of truth for current behavior, the log is the audit trail.
